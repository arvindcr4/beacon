**Full description** (max 4,000 chars)
```
Beacon is a universal email client built around an AI that actually triages.
Connect Gmail, Office 365, Yahoo, AOL, iCloud, or any IMAP mailbox — Beacon
gives you one mobile-first inbox across all of them, with a single sentence
of context on every message and a 1-click draft reply in your tone.

WHAT MAKES IT DIFFERENT

• Unified inbox across every provider. Switch accounts with a chip tap.
• Per-message priority — Now / Soon / Later — generated by Claude Haiku 4.5,
  with a one-sentence reason on the inbox card.
• AI summary on every message. No more reading three paragraphs to find
  the ask.
• Draft a reply in five tones (neutral, warm, direct, apologetic,
  enthusiastic) with streaming tokens from Claude Sonnet 4.6. Optional
  intent: "decline politely", "ask for the slides".
• Compose new email by telling Beacon the intent — it drafts, you edit.
• Real IMAP support with secure app passwords for Yahoo, AOL, iCloud,
  Fastmail, or custom servers.
• OAuth for Gmail (mail.google.com scope) and Microsoft 365
  (Mail.ReadWrite + Mail.Send).
• Search, labels, archive, delete — unified across all accounts.
• Installable PWA. Offline shell. Mobile-first design.

PRIVACY

Beacon is built around an AES-256-GCM credential vault. Your mailbox
passwords and OAuth refresh tokens are encrypted on the server before
storage and never returned to the client. No analytics, no ads, no
third-party trackers. AI calls are routed only to Anthropic, only for the
features you trigger, never for training. See the full privacy policy at
beacon-dun-one.vercel.app/privacy.

REQUIREMENTS

Android 5.0+ (API 21). The app loads the live PWA at
beacon-dun-one.vercel.app — content updates ship automatically with every
deploy.

BUILT WITH CLAUDE CODE

Beacon was built end-to-end with Claude Code's multi-agent workflow,
specs-driven development, custom subagents, slash-skills, and PreToolUse /
PostToolUse hooks. Source: github.com/arvindcr4/beacon.
```

## App access

Play reviewers need to actually be able to log in to test the app.
Use the "All functionality is available without special access" radio only
if your test reviewers can sign up with any email. Beacon supports email-only
sign-in (no password), so:

| Field | Value |
|-------|-------|
| Demo credentials needed? | No — testers can sign up with any email on /signin |
| Instructions for reviewers | Sign in with any email. Add an IMAP mailbox to see the inbox (use a Yahoo/AOL test account with an app password). Or wait for OAuth approval to test Gmail. |
